# AI任务集市 - 快速启动指南

## 🚀 快速启动

### 1. 打开两个命令行窗口

#### 窗口1：启动后端
```bash
cd d:\aishop
npm run dev:backend
```

#### 窗口2：启动前端
```bash
cd d:\aishop
npm run dev:frontend
```

### 2. 访问应用

打开浏览器访问：http://localhost:5173

### 3. 登录测试账户

| 角色 | 用户名 | 密码 | 说明 |
|------|--------|------|------|
| 主管理员 | admin | admin123 | 可审核任务、分配任务 |
| 技术专家 | expert | expert123 | 可审核任务 |
| 普通用户 | user | user123 | 可发布、申领任务 |

## 🔍 调试工具

如果任务列表看不到，请按顺序尝试：

### 方法1：使用API测试页面
访问：http://localhost:5173/api-test.html

### 方法2：使用内置调试页面
登录后访问：http://localhost:5173/debug

### 方法3：查看浏览器控制台
按F12 → Console标签

## ✅ 预期结果

### 普通用户登录后
- 首页应显示7个已发布的任务
- 可以申领任务
- 可以发布新任务

### 管理员/专家登录后
- 可以看到所有任务（包括待审核的）
- 可以审核任务（待审核的任务会有审核按钮）
- 可以分配任务给申领人
- 可以查看审批历史记录

## 📝 功能说明

### 任务审批记录
- 在任务详情页可以看到完整的审批历史
- 显示审批人、审批时间、审批意见
- 显示三个维度的评分（应用思维、产品思维、工程思维）

### 悬赏类型
- 可以单选澳维豆或悬赏物
- 也可以同时选择两者

## 🛠️ 常见问题

### Q: 页面显示"暂无任务"？
**A:** 
1. 检查后端是否启动（访问 http://localhost:3000/api/health）
2. 使用调试页面测试API
3. 查看浏览器控制台错误

### Q: 登录失败？
**A:**
1. 检查后端是否运行
2. 确认用户名密码正确
3. 清除浏览器LocalStorage后重试

### Q: 数据库没有任务数据？
**A:**
1. 删除 `d:\aishop\data\database.sqlite`
2. 重启后端，会自动创建测试数据
